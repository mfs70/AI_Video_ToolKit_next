# Editor UX: keyframe-safe export, markers, undo/remove, segment preview

## 1) Что значит: `-c copy` vs keyframe-aware split fallback

### `-c copy`
- FFmpeg не перекодирует поток, а копирует исходные аудио/видео пакеты в новый контейнер.
- Плюсы: очень быстро, без потери качества, минимальная нагрузка CPU.
- Минусы: точность резки ограничена ключевыми кадрами (keyframes / I-frames).

Если разрез начинается не с keyframe, возможны:
- сдвиг старта на ближайший keyframe,
- черный/битый старт первых кадров,
- рассинхрон или артефакты в начале сегмента.

### Keyframe-aware split fallback
Идея: сначала пытаться резать `-c copy`, но если точность/качество сегмента плохая — автоматически включать безопасный режим.

Безопасный режим:
- либо сдвигать старт/конец к ближайшим keyframe (быстро, но менее точно),
- либо перекодировать только «проблемные» сегменты (точно, но медленнее).

Практический алгоритм:
1. Пробуем сегмент через `-c copy`.
2. Валидация сегмента (`ffprobe`, проверка duration/первых кадров).
3. Если сегмент невалиден/неточен:
   - fallback A: snap к keyframe,
   - fallback B: re-encode сегмента (`libx264` / `aac`).

## 2) Что значит: undo/remove marker

### Назначение
При монтаже пользователь часто экспериментирует с точками. Нужны быстрые операции:
- отменить последнюю постановку маркера,
- удалить конкретный маркер,
- очистить все маркеры выбранного типа.

### Какие операции нужны в UX
- `Ctrl+Z`: Undo последнего действия с маркером.
- `Delete`/контекстное меню: удалить выбранный marker.
- Кнопки:
  - Remove Input,
  - Remove Output,
  - Remove Last Cut,
  - Clear Cuts.

### Технически
- Стек действий `MarkerAction` (Add/Remove/Move, type, value).
- Undo/Redo как в редакторе: два стека (undo/redo).

## 3) “Полный редакторский UX”: что включаем следующим шагом

### Segment Preview Panel
- Список сегментов в правой панели:
  - № сегмента,
  - Start / End,
  - Duration,
  - тип экспорта (copy/reencode fallback).
- Кнопки:
  - Rebuild Segments,
  - Preview Selected Segment,
  - Export Selected,
  - Export All.

### Безопасный экспорт
- Экспортный пайплайн:
  1. Build segments from I/O/Cut markers.
  2. Для каждого сегмента пробуем `-c copy`.
  3. Проверяем валидность/точность.
  4. При проблеме -> fallback policy.
  5. Логируем решение по каждому сегменту.

### UX-правила маркеров
- Input один, Output один, Cuts много.
- Cuts строго внутри диапазона [Input, Output] при их наличии.
- Нельзя пересекать/дублировать маркеры.
- Визуальная подсветка ошибок (tooltip + logger).

## 4) План реализации (технический)

1. Добавить модель данных:
   - `TimelineMarkers`, `Segment`, `ExportPolicy`, `MarkerAction`.
2. Добавить сервис:
   - `SegmentBuilderService`,
   - `SafeExportService` (copy + fallback).
3. Расширить `MainWindow`:
   - DataGrid/ListView сегментов,
   - кнопки preview/export/undo/remove.
4. Расширить `TimelineControl`:
   - hit-test/select marker,
   - визуал selected marker.
5. Логи:
   - протокол экспорта по каждому сегменту.

## 5) Критерии готовности
- После постановки маркеров пользователь видит корректный список сегментов.
- Preview любого сегмента стартует с его начала.
- Export All создаёт ожидаемые части в `Cut`.
- Проблемные сегменты автоматически переходят на fallback без падения процесса.
- Undo/remove marker работают предсказуемо.
