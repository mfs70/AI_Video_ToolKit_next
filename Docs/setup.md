🧰 2. Установка .NET SDK
2.1 Выбор версии
Используем:
👉 .NET 8 LTS (x64)
Почему:
•	стабильная долгосрочная поддержка (LTS)
•	максимальная совместимость с WPF
•	меньше багов, чем у новых версий
•	подходит для production







________________________________________
2.2 Установка
1.	Скачать .NET 8 SDK (x64) 
https://dotnet.microsoft.com/en-us/download/dotnet/thank-you/sdk-8.0.420-windows-x64-installer
https://learn.microsoft.com/en-us/dotnet/core/tools/dotnet-install-script
https://github.com/dotnet/install-scripts
•	Скрипт не устанавливает переменную окружения DOTNET_ROOT и не добавляет расположение установки в переменную PATH пользователя — эти изменения нужно выполнить вручную.
•	Для настройки среды разработки или запуска приложений рекомендуется использовать официальные установщики, а не скрипты.
2.	Запустить установщик
3.	Установить с настройками по умолчанию
________________________________________
2.3 Проверка
Открыть терминал:
dotnet --version
Ожидаемый результат:
8.0.xxx
________________________________________
2.4 Если установлен .NET 10
Можно оставить, но зафиксировать версию проекта через файл:
global.json (в корне проекта)
{
  "sdk": {
    "version": "8.0.100"
  }
}

🔧 5. Установка Git
________________________________________
5.1 Скачивание
Скачать Git for Windows (64-bit)
https://git-scm.com/install/windows
или в PowerShell выполнить команду: winget install --id Git.Git -e --source winget

________________________________________
5.2 Ключевые настройки при установке
Шаг	Выбор
Editor	Visual Studio Code
Branch	main
PATH	Git from command line
HTTPS	OpenSSL
Line endings	Windows → Unix
Terminal	Windows console
Credential helper	Git Credential Manager
________________________________________
5.3 Проверка
git --version

🧠 11. Рекомендации уровня production
________________________________________
1. Всегда фиксируй версию .NET
→ через global.json
________________________________________
2. Работай из корня проекта
→ чтобы корректно работали пути
________________________________________
3. Не храни бинарники в Git
→ _Portable_ всегда вне репозитория
________________________________________
4. Делай частые коммиты
→ маленькие изменения = легче отлаживать
________________________________________
5. Пиши осмысленные сообщения коммитов
Пример:
Add FFmpeg probe module
Fix UI layout
Implement async logging
